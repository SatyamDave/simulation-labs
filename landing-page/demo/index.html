      03 — How it works
      
We don't prompt an agent to act rushed.
We break the machine by measured amounts.

      
Each agent is a real computer-use model driving a real browser. We mechanically degrade what it sees and how it clicks — coordinate noise on every tap, tight patience budgets, small phone viewports, downscaled frames — to model how real segments behave. Seeded and reproducible: same tremor σ, same budget, same viewport every run. Not vibes.

      05 — Why it blocked it
      
Green build. Passing tests. Users who can't finish.

      
Imprecise-tap users are ~15–20% of mobile signups. On a 24px target, most now fail — ~1,600/mo of ~8,000 that would have converted, gone.
A completion drop concentrated in one segment, on a flow that passed every unit test. No error, no alert — just fewer accounts.
