SANITY vs DEMO_PLAYBOOK.md: full-8 worst-case = 41 min matches its "~40 min for 8x30" (its 8x30=48 min shorthand; real max_steps sum =205 calls gives 41 min -- close agreement). The playbook's "~3-6 min for 4" is its THREE fast-failing personas (grandma-72, impatient-mobile, low-vision) which complete/abandon early; this preset's demo-4 swaps in power-user (a full-40 baseline), so its worst-case rises to ~17 min / ~10 min expected -- same order, higher because power-user runs long. SIM-CLOCK INDEPENDENCE: per-step simulated cost = _THINK_TIME_S(4.0) + page(~1s) = 5s, so a persona hits TIME_BUDGET after deadline_s/per_step steps REGARDLESS of RPM/API latency (Holo queue is excluded from patience, commit 416e488). E.g. impatient-mobile (deadline_s=30) abandons after ~6 steps whether the API runs at 5 or 60 RPM -- RPM only changes how long WE wait for those steps, never the verdict. THROUGHPUT: steps/minute == RPM by construction.
